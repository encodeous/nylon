title: CLI Reference
description: Commands and flags for the nylon command-line interface.
---

The `nylon` binary is both the daemon entrypoint and the operator CLI. Commands that inspect or change a running daemon use Nylon IPC over the WireGuard UAPI socket.

By default, daemon IPC commands connect to interface `nylon`. If your node config uses a different `interface_name`, pass `--interface` or `-i`.

```bash
nylon status --interface nylon0
```

## Daemon

### `nylon run`

Starts the local Nylon node.

```bash
nylon run --config central.yaml --node node.yaml
```

Flags:

| Flag | Default | Description |
| --- | --- | --- |
| `-c, --config` | `./central.yaml` | Central network config path. |
| `-n, --node` | `./node.yaml` | Local node config path. |
| `-l, --log` | empty | Log file path. Overrides `log_path` from node config. |
| `-v, --verbose` | `false` | Enables debug logging. |
| `-j, --json` | `false` | Emits structured JSON logs. |
| `--dbg-probe` | `false` | Logs probe traffic. |
| `--dbg-wg` | `false` | Logs WireGuard internals. |
| `--dbg-repo` | `false` | Logs config distribution update checks. |
| `--dbg-perf` | `false` | Starts the pprof server on port `6060`. |
| `--dbg-trace` | `false` | Writes Go runtime trace data to `trace.out`. |
| `--dbg-trace-tc` | `false` | Enables packet routing trace events for `nylon trace`. |

## Runtime Inspection

These commands require a running Nylon daemon.

Common flags:

| Flag | Default | Description |
| --- | --- | --- |
| `-i, --interface` | `nylon` | WireGuard/Nylon interface name. |
| `--json` | `false` | Print the raw IPC response as protojson. |

### `nylon status`

Shows the running node, neighbors, WireGuard counters, and endpoint health. The CLI always fetches the full IPC status snapshot and filters the human-readable output locally.

```bash
nylon status -i nylon0
nylon status -i nylon0 --routes
nylon status -i nylon0 --full
nylon status -i nylon0 --json
```

Flags:

| Flag | Default | Description |
| --- | --- | --- |
| `--routes` | `false` | Include neighbor route advertisements and selected/forward/exit route tables in text output. |
| `--full` | `false` | Include route details plus seqnos, feasibility distances, WireGuard endpoints, and advertised routes. |
| `--no-color` | `false` | Disable ANSI color in text output. Color is also disabled when stdout is not a terminal or `NO_COLOR` is set. |

Default text output includes:

| Section | Contents |
| --- | --- |
| Node | Node id, interface name, public key, listen port, config timestamp, trace state, and aggregate stats. |
| Neighbors | Public key, passive-client state, best metric, WireGuard endpoint and transfer counters, and all Nylon endpoints for each neighbor. |
| Advertised | Local advertised prefixes and passive-hold state. |

`--routes` also prints selected routes and the forward/exit tables. `--full` adds Babel seqnos, feasibility distances, route expiry, retractions, neighbor route advertisements, and per-endpoint RTT values.

`--json` prints the raw `IpcResponse` protojson from the daemon. It is not filtered by `--routes` or `--full`.

### `nylon probe <peer>`

Immediately probes all known endpoints for a neighbor.

```bash
nylon probe bob -i nylon0
```

This sends Nylon probe packets and reports whether each endpoint probe was successfully queued. Link state updates asynchronously when responses arrive.

### `nylon trace`

Streams live packet-routing trace events.

```bash
nylon run --dbg-trace-tc
nylon trace -i nylon0
```

Tracing must be enabled when the daemon starts with `--dbg-trace-tc`.

### `nylon reload --file <central.yaml>`

Applies a central config file to a running daemon without restarting.

```bash
nylon reload -i nylon0 --file ./central.yaml
```

The daemon validates and applies the config on Nylon's main dispatch loop. Results are:

| Result | Meaning |
| --- | --- |
| `NOOP` | Config is equivalent to the current state. |
| `APPLIED` | Config was accepted and applied. |
| `REJECTED` | Config was invalid or failed to apply. |
| `RESTART_REQUIRED` | Config cannot be applied live. |

## Configuration

### `nylon verify <config>`

Validates a config file using Nylon's built-in validators.

```bash
nylon verify ./central.yaml
nylon verify --node ./node.yaml
```

By default, `verify` treats the input as a central config. Use `--node` for local node config validation.

### `nylon seal`

Signs and encrypts a central config into a distribution bundle.

```bash
nylon seal -c central.yaml -k central.key -o central.nybundle
```

Flags:

| Flag | Default | Description |
| --- | --- | --- |
| `-c, --config` | `./central.yaml` | Central config path. |
| `-k, --key` | `./central.key` | Distribution private key path. |
| `-o, --output` | `./central.nybundle` | Bundle output path. |

### `nylon open <central-public-key>`

Decrypts and verifies a distribution bundle, then prints the parsed central config.

```bash
nylon open "$(cat central.pub)" -i central.nybundle
```

Flags:

| Flag | Default | Description |
| --- | --- | --- |
| `-i, --input` | `./central.nybundle` | Bundle input path. |

## Keys and Version

### `nylon key`

Generates a Nylon keypair. The private key is written to stdout and the public key is written to stderr.

```bash
nylon key > node.key 2> node.pub
```

### `nylon genkey`

Generates a private key only.

```bash
nylon genkey > node.key
```

### `nylon pubkey`

Reads a private key from stdin and prints its public key.

```bash
nylon pubkey < node.key > node.pub
```

### `nylon version`

Prints build version information.

```bash
nylon version
```
