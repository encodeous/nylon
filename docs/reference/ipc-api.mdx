---
title: IPC API
description: JSON/protobuf API for controlling a running Nylon daemon.
---

Nylon exposes a developer-facing IPC API over the same WireGuard UAPI socket used by `wg`. The payload format is newline-delimited JSON encoded with protobuf `protojson`.

The protobuf schema is defined in `protocol/nylon_ipc.proto`. The CLI uses the same API and only renders responses for humans; the daemon protocol itself is JSON.

## Transport

Connect to the WireGuard UAPI socket for the Nylon interface.

On Linux, the default path is:

```text
/var/run/wireguard/<interface>.sock
```

For example, if the interface is `nylon0`:

```text
/var/run/wireguard/nylon0.sock
```

Send a custom UAPI operation followed by one single-line protojson request:

```text
get=nylon
{"status":{}}
```

The daemon replies with one single-line protojson `IpcResponse`:

```json
{"ok":true,"error":"","status":{"node":{"nodeId":"alice","interface":"nylon0","publicKey":"...","listenPort":51820,"configTimestamp":"1740832962209309000","traceEnabled":false,"stats":{"neighbourCount":2,"activeEndpointCount":3,"selectedRouteCount":4,"advertisedPrefixCount":1,"txBytes":"1234","rxBytes":"5678"}},"neighbours":[{"peerId":"bob","publicKey":"...","passiveClient":false,"bestMetric":96,"endpoints":[{"address":"bob.example:51820","resolved":"203.0.113.10:51820","active":true,"remoteInit":false,"metric":96,"isBest":true}]}],"routes":{"selected":[],"forward":[],"exit":[]},"feasibilityDistances":[]}}
```

Nylon IPC responses do not include the WireGuard UAPI `errno=0` trailer. Every line returned by a Nylon IPC request is protojson.

## Request Envelope

`IpcRequest` contains exactly one request payload.

```proto
message IpcRequest {
  oneof request {
    StatusRequest status = 1;
    ProbeRequest probe = 5;
    ReloadRequest reload = 6;
    TraceRequest trace = 7;
  }
}
```

Protojson uses lowerCamelCase field names by default. For example, `peer_id` is encoded as `peerId`.

## Response Envelope

Every non-streaming request returns one `IpcResponse`.

```proto
message IpcResponse {
  bool ok = 1;
  string error = 2;
  oneof response {
    StatusResponse status = 3;
    ProbeResponse probe = 7;
    ReloadResponse reload = 8;
    TraceEvent trace = 9;
  }
}
```

If `ok` is `false`, `error` contains a human-readable error string and the response payload may be absent.

## Methods

### Status

Request:

```json
{"status":{}}
```

The status method always returns the complete status snapshot. Clients should request `status` once and filter locally.

Top-level payload:

| Field | Type | Description |
| --- | --- | --- |
| `node` | `NodeStatus` | Local node identity, configuration, advertisements, seqnos, and aggregate counters. |
| `neighbours` | array of `NeighbourInfo` | Router neighbors and passive clients known to this node. A neighbor may contain multiple endpoints. |
| `routes` | `RouteTables` | Babel selected routes plus data-plane forward and exit tables. |
| `feasibilityDistances` | array of `FeasibilityDistance` | Feasibility distance per route source. |

`NodeStatus`:

| Field | Type | Description |
| --- | --- | --- |
| `nodeId` | string | Local node id. |
| `interface` | string | Active interface name. |
| `publicKey` | string | Local Nylon public key. |
| `listenPort` | uint32 | WireGuard listen port. |
| `configTimestamp` | int64 string | Current central config timestamp. |
| `traceEnabled` | bool | Whether packet trace events are enabled. |
| `advertised` | array of `Advertisement` | Prefixes currently advertised by this node. |
| `seqnos` | array of `SeqnoEntry` | Local Babel seqno per advertised prefix. |
| `stats` | `NodeStats` | Aggregate neighbor, route, advertisement, and transfer counters. |

`NeighbourInfo`:

| Field | Type | Description |
| --- | --- | --- |
| `peerId` | string | Neighbor node id. |
| `publicKey` | string | Neighbor Nylon/WireGuard public key. |
| `passiveClient` | bool | Whether this peer is a passive client. |
| `bestMetric` | uint32 | Metric of the best active endpoint, or `INF` if no endpoint is active. |
| `endpoints` | array of `EndpointInfo` | All known Nylon endpoints for this neighbor. |
| `routes` | array of `NeighRoute` | Routes advertised by this neighbor. |
| `advertised` | array of `Advertisement` | Prefixes this node is currently advertising on behalf of the neighbor. |
| `wireguard` | `WireGuardPeerStats` | WireGuard handshake, transfer, keepalive, and active endpoint data. |

`EndpointInfo`:

| Field | Type | Description |
| --- | --- | --- |
| `address` | string | Configured or discovered endpoint value. |
| `resolved` | string | Resolved `ip:port`, or empty if unresolved. |
| `active` | bool | Whether the endpoint is active. |
| `remoteInit` | bool | Whether the endpoint was discovered from remote traffic. |
| `metric` | uint32 | Current endpoint metric. |
| `isBest` | bool | Whether this is the best endpoint for the neighbor. |
| `filteredRttNs` | int64 string | Filtered RTT estimate in nanoseconds. |
| `stabilizedRttNs` | int64 string | Stabilized RTT estimate in nanoseconds. |

Route model:

| Field | Type | Description |
| --- | --- | --- |
| `Source.nodeId` | string | Origin node id for a route source. |
| `Source.prefix` | string | Origin prefix. |
| `FD.seqno` | uint32 | Babel sequence number. |
| `FD.metric` | uint32 | Babel route metric. |
| `PubRoute.source` | `Source` | Route source. |
| `PubRoute.fd` | `FD` | Feasibility-distance tuple. |
| `NeighRoute.pubRoute` | `PubRoute` | Route published by a neighbor. |
| `NeighRoute.expireAtUnix` | int64 string | Expiry time as Unix seconds. |
| `SelRoute.pubRoute` | `PubRoute` | Selected route source and FD. |
| `SelRoute.nh` | string | Selected next-hop node id. |
| `SelRoute.expireAtUnix` | int64 string | Expiry time as Unix seconds. |
| `SelRoute.retractedBy` | array of string | Neighbors that retracted the held route. |
| `RouteTableEntry.prefix` | string | Forward or exit table prefix. |
| `RouteTableEntry.nh` | string | Forward or exit next-hop node id. |
| `RouteTableEntry.blackhole` | bool | Whether this data-plane entry blackholes traffic. |

`Advertisement`:

| Field | Type | Description |
| --- | --- | --- |
| `nodeId` | string | Node id whose prefix is being advertised. |
| `prefix` | string | CIDR prefix. |
| `metric` | uint32 | Current advertised metric. |
| `expiryUnix` | int64 string | Expiry time as Unix seconds. |
| `passiveHold` | bool | Whether this is a passive-hold advertisement. |

### Probe

Request:

```json
{"probe":{"peerId":"bob"}}
```

Response payload:

| Field | Type | Description |
| --- | --- | --- |
| `results` | array | Per-endpoint probe enqueue results. |
| `results[].address` | string | Endpoint value. |
| `results[].success` | bool | Whether the probe packet was queued successfully. |
| `results[].error` | string | Error message if `success` is false. |

Probe results report whether Nylon could send the probe. Endpoint health changes asynchronously when responses are received.

### Reload

Request:

```json
{"reload":{"file":"/app/config/central.yaml"}}
```

The path is resolved on the daemon host/container, not on the client process.

Response payload:

| Field | Type | Description |
| --- | --- | --- |
| `result` | enum | `NOOP`, `APPLIED`, `REJECTED`, or `RESTART_REQUIRED`. |
| `message` | string | Additional error or status text. |

### Trace

Request:

```json
{"trace":{}}
```

If tracing is enabled, the daemon first returns:

```json
{"ok":true,"error":""}
```

Then it streams newline-delimited `IpcResponse` messages with `trace` payloads:

```json
{"ok":true,"error":"","trace":{"line":"..."}}
```

Tracing requires the daemon to be started with:

```bash
nylon run --dbg-trace-tc
```

If tracing is disabled, the initial response has `ok:false`.

## Raw Client Example

This Python example sends a status request directly to the UAPI socket:

```python
import socket

sock = socket.socket(socket.AF_UNIX, socket.SOCK_STREAM)
sock.connect("/var/run/wireguard/nylon0.sock")
sock.sendall(b'get=nylon\n{"status":{}}\n')
print(sock.recv(65535).decode(), end="")
sock.close()
```

## Error Handling

Malformed JSON, unknown request types, invalid peers, rejected reloads, and disabled tracing are reported as protojson `IpcResponse` messages with `ok:false`.

Transport-level errors, such as a missing socket or permission failure, occur before a valid `IpcResponse` exists and must be handled by the client.
